# Roadmap

This project follows a staged safety-first roadmap.

## Complete

- Phase 1: scaffold
- Phase 2: YAML-driven playback
- Phase 3: expanded controller actions
- Phase 4: macro quality-of-life
- Phase 5: profiles and examples
- Phase 6: safety and reliability
- Phase 7: perception foundation
- Phase 8: image diagnostics

## Next

- Phase 9: named screen regions
- Phase 10: region catalogs by layout/profile
- Phase 11: visual region overlays
- Phase 12: baseline snapshot drift checks
- Phase 13: threshold-only screen checks
- Phase 14: template matching diagnostics
- Phase 15: conditional wait diagnostics
- Phase 16: guarded action gates

## Guardrails

- Keep all automation offline and user-initiated.
- Preserve dry-run and validation-first workflow.
- Keep perception tools diagnostic until explicitly promoted.
- Maintain explicit stop controls and runtime limits.

For detailed rationale and sequencing, see `docs/AUDIT_AND_ROADMAP.md`.
